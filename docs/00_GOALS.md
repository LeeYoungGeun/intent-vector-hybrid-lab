# 목표(Goals)

## 무엇을 검증하나
이 템플릿은 “의도 기반 벡터/하이브리드 검색” 파이프라인에서 **혼동(예: 욕실 매트 vs 요가 매트)**, **누락**, **불필요한 후보 과다**가 발생하는 지점을 정량적으로 찾고, 벤더/모델/파라미터 조합을 비교하는 목적입니다.

검증 대상 단계:
- (1) Embedding
- (2) Vector Retrieval (ANN/VectorDB)
- (3) Hybrid Fusion (BM25 + Vector)
- (4) Re-ranking
- (5) Score/Rule Filtering

## 산출물
- 벤더별/파이프라인별 결과 리포트(요약 + 실패 케이스 Top N)
- 단계별 지연시간/비용(가능하면 토큰/호출수 포함) 비교표
- 애매성/오탐 유형별 개선 가이드(룰 vs 프롬프트 vs 모델 교체)

## 비목표(Non-goals)
- 운영 서비스(프로덕션) 배포
- UI 완성
- 결제/인증 등 비핵심 도메인
